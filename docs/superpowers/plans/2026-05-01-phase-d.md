# Phase D Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close customer-trust gaps: NDA gate, Type C short-answer questionnaire, "cleaned-up" dashboard banner, and stop test fixtures from leaking into the live operator's email.

**Architecture:** Single migration `0011_phase_d.sql` adds two columns to `customers` and one new table `customer_questions`. New middleware enforces the NDA gate via strict allowlist. Type C questionnaire ships as a small domain (`domain/customer-questions/{repo,service}.js`) with admin-create + customer-answer routes. "Cleaned-up" banner reads existing `audit_log` rows. Test-pollution cleanup ships as a shared helper retrofitted into all integration tests that touch digest/outbox tables, plus a one-time historical-cleanup script.

**Tech Stack:** Node.js 20 (per-project nvm), Fastify, Kysely + raw SQL, EJS views, Vitest, PostgreSQL 16, MailerSend (transactional via `email_outbox` worker).

**Spec:** `docs/superpowers/specs/2026-05-01-phase-d-design.md`.

---

## Conventions used throughout this plan

- **Tests:** Vitest. Integration tests gate on `RUN_DB_TESTS=1` and run via `sudo bash scripts/run-tests.sh`. Use a per-test `tag = '<feature>_test_${Date.now()}'` and clean up in `afterAll`.
- **Routes:** live in `routes/<surface>/<area>.js` (NOT `src/routes/...` — the spec wrote `src/routes` but the codebase uses `routes/`). Each route module exports `register*Routes(app)` and is wired in `routes/_index.js` (admin) or directly in `server.js`.
- **Audit writes:** `writeAudit(tx, {...})` from `lib/audit.js`. Pass `actorType`, `actorId`, `action`, `targetType`, `targetId`, `metadata` (object), `visibleToCustomer`, `ip`, `userAgentHash`. The `baseAudit(ctx)` helper in each domain produces `{ ip, userAgentHash, metadata: {...} }` from the Fastify request `ctx`.
- **Digest writes:** `recordForDigest(tx, { recipientType, recipientId, customerId, bucket, eventType, ... })` from `domain/digest/recorder.js` (or wherever Phase B placed it — grep first).
- **Email enqueue:** insert into `email_outbox` with `idempotency_key`, `to_address`, `template`, `locale`, `locals`. The outbox worker picks it up.
- **Commits:** `feat(phase-d-N): subject` / `test(phase-d-N): subject` / `fix(phase-d-N): subject` matching the existing `phase-b-*` convention.
- **Run a single failing test:** `sudo bash scripts/run-tests.sh tests/integration/<path>/<file>.test.js -t '<test name pattern>'`.

```javascript
import { sql } from 'kysely';

// Cleanup helper for test rows that survive customer/admin teardown via
// non-CASCADEd referential paths. The 2026-04-30 fixture leak proved the
// digest and outbox tables can outlive their fixture customer/admin rows
// and get processed by the live workers — this helper closes the gap.
//
// Inputs (all optional; helper is a no-op when none are given):
//   - recipientIds:    UUIDs of customer_users / admins whose digest rows
//                      and schedules should be deleted.
//   - emailAddresses:  exact to_address values to delete from email_outbox.
//   - e2eEmailPattern: if true, also delete email_outbox rows whose
//                      to_address matches '%_e2e@example.com' (defense-
//                      in-depth fallback for fixtures that didn't tag
//                      their emails into emailAddresses).
export async function pruneTestPollution(db, {
  recipientIds = [],
  emailAddresses = [],
  e2eEmailPattern = false,
} = {}) {
  if (recipientIds.length > 0) {
    await sql`
      DELETE FROM pending_digest_items
      WHERE recipient_id = ANY(${recipientIds}::uuid[])
    `.execute(db);
    await sql`
      DELETE FROM digest_schedules
      WHERE recipient_id = ANY(${recipientIds}::uuid[])
    `.execute(db);
  }
  if (emailAddresses.length > 0) {
    await sql`
      DELETE FROM email_outbox
      WHERE to_address = ANY(${emailAddresses}::text[])
    `.execute(db);
  }
  if (e2eEmailPattern) {
    await sql`
      DELETE FROM email_outbox
      WHERE to_address ILIKE ${'%\\_e2e@example.com'} ESCAPE '\\'
    `.execute(db);
  }
}
```

```javascript
#!/usr/bin/env node
// One-time recovery: deletes test-fixture-flavoured rows that survived
// previous test runs and are sitting in the live DB. Patterns target the
// 2026-04-30 incident shape:
//   - email_outbox.to_address ILIKE '%_e2e@example.com'
//   - pending_digest_items.recipient_id pointing to customer_users or
//     admins whose email LIKE '%_test_%' or LIKE '%_e2e@%'
//   - digest_schedules.recipient_id same predicate
//
// Default mode is dry-run (counts + sample row IDs printed). Pass --apply
// to actually delete inside a single transaction with rollback on error.
import { sql } from 'kysely';
import { createDb } from '../config/db.js';
import { loadEnv } from '../config/env.js';

const apply = process.argv.includes('--apply');
const env = loadEnv();
const db = createDb({ connectionString: env.DATABASE_URL });

const E2E_PATTERN = '%\\_e2e@example.com';
const TEST_PATTERN = '%\\_test\\_%';

async function reportCounts() {
  const outboxR = await sql`
    SELECT COUNT(*)::int AS n FROM email_outbox
    WHERE to_address ILIKE ${E2E_PATTERN} ESCAPE '\\'
       OR to_address ILIKE ${TEST_PATTERN} ESCAPE '\\'
  `.execute(db);
  const digestR = await sql`
    SELECT COUNT(*)::int AS n FROM pending_digest_items
    WHERE recipient_id IN (
      SELECT id FROM customer_users WHERE email ILIKE ${E2E_PATTERN} ESCAPE '\\' OR email ILIKE ${TEST_PATTERN} ESCAPE '\\'
      UNION ALL
      SELECT id FROM admins WHERE email ILIKE ${E2E_PATTERN} ESCAPE '\\' OR email ILIKE ${TEST_PATTERN} ESCAPE '\\'
    )
  `.execute(db);
  const schedR = await sql`
    SELECT COUNT(*)::int AS n FROM digest_schedules
    WHERE recipient_id IN (
      SELECT id FROM customer_users WHERE email ILIKE ${E2E_PATTERN} ESCAPE '\\' OR email ILIKE ${TEST_PATTERN} ESCAPE '\\'
      UNION ALL
      SELECT id FROM admins WHERE email ILIKE ${E2E_PATTERN} ESCAPE '\\' OR email ILIKE ${TEST_PATTERN} ESCAPE '\\'
    )
  `.execute(db);
  console.log('Stale row counts:');
  console.log('  email_outbox:          ', outboxR.rows[0].n);
  console.log('  pending_digest_items:  ', digestR.rows[0].n);
  console.log('  digest_schedules:      ', schedR.rows[0].n);
}

async function applyDeletes() {
  await db.transaction().execute(async (tx) => {
    const taggedIdsR = await sql`
      SELECT id FROM customer_users WHERE email ILIKE ${E2E_PATTERN} ESCAPE '\\' OR email ILIKE ${TEST_PATTERN} ESCAPE '\\'
      UNION ALL
      SELECT id FROM admins WHERE email ILIKE ${E2E_PATTERN} ESCAPE '\\' OR email ILIKE ${TEST_PATTERN} ESCAPE '\\'
    `.execute(tx);
    const recipientIds = taggedIdsR.rows.map(r => r.id);

    const o = await sql`
      DELETE FROM email_outbox
      WHERE to_address ILIKE ${E2E_PATTERN} ESCAPE '\\'
         OR to_address ILIKE ${TEST_PATTERN} ESCAPE '\\'
      RETURNING id
    `.execute(tx);
    console.log(`deleted email_outbox: ${o.rows.length}`);

    if (recipientIds.length > 0) {
      const d = await sql`
        DELETE FROM pending_digest_items WHERE recipient_id = ANY(${recipientIds}::uuid[])
        RETURNING id
      `.execute(tx);
      const s = await sql`
        DELETE FROM digest_schedules WHERE recipient_id = ANY(${recipientIds}::uuid[])
        RETURNING id
      `.execute(tx);
      console.log(`deleted pending_digest_items: ${d.rows.length}`);
      console.log(`deleted digest_schedules: ${s.rows.length}`);
    }
  });
}

await reportCounts();
if (apply) {
  console.log('--apply set; performing deletes inside a single transaction');
  await applyDeletes();
} else {
  console.log('Dry run only. Re-run with --apply to perform deletes.');
}
await db.destroy();
```

```sql
ALTER TABLE customers
  ADD COLUMN nda_signed_at TIMESTAMPTZ NULL;

ALTER TABLE customers
  ADD COLUMN last_cleanup_banner_dismissed_at TIMESTAMPTZ NULL;

CREATE TABLE customer_questions (
  id                            UUID PRIMARY KEY,
  customer_id                   UUID NOT NULL REFERENCES customers(id) ON DELETE CASCADE,
  created_by_admin_id           UUID NOT NULL REFERENCES admins(id),
  answered_by_customer_user_id  UUID NULL REFERENCES customer_users(id),
  question                      TEXT NOT NULL,
  answer_text                   TEXT NULL,
  status                        TEXT NOT NULL DEFAULT 'open'
    CHECK (status IN ('open', 'answered', 'skipped')),
  created_at                    TIMESTAMPTZ NOT NULL DEFAULT now(),
  answered_at                   TIMESTAMPTZ NULL
);

CREATE INDEX customer_questions_customer_id_status_idx
  ON customer_questions (customer_id, status);
```

- [ ] **Step 3: Implement the middleware**

```javascript
// Strict allowlist for routes a customer may access while their NDA is
// not yet recorded. Anything not on the list is gated.
const PAGE_ALLOW = [
  /^\/customer\/waiting$/,
  /^\/customer\/profile(\/.*)?$/,
  /^\/auth\/logout$/,
  /^\/auth\/email-change\/.*/,
  /^\/auth\/password-reset\/.*/,
];

const API_ALLOW = [
  /^\/api\/customer\/me$/,
  /^\/api\/customer\/profile(\/.*)?$/,
  /^\/api\/auth\/.*/,
];

export async function ndaGate(req, reply) {
  // req.customer is populated by the customer-session loader earlier in
  // the request lifecycle. If absent, we're on a non-customer route and
  // should not be running this middleware at all — but defend anyway.
  if (!req.customer) return;
  if (req.customer.nda_signed_at) return;

  const path = (req.raw?.url ?? req.url ?? '').split('?')[0];

  if (path.startsWith('/api/')) {
    if (API_ALLOW.some(re => re.test(path))) return;
    reply.status(403).send({ error: 'nda_required' });
    return;
  }

  if (PAGE_ALLOW.some(re => re.test(path))) return;
  reply.redirect('/customer/waiting', 302);
}
```

```javascript
import { sql } from 'kysely';
import * as repo from './repo.js';
import { writeAudit, baseAudit } from '../../lib/audit.js';
import { recordForDigest } from '../digest/recorder.js'; // adjust path if Phase B placed it elsewhere
import { listActiveCustomerUsers } from '../customer-users/repo.js'; // grep — may live elsewhere
import { listAdmins } from '../admins/repo.js'; // grep

export async function createQuestion(db, { customerId, createdByAdminId, question }, ctx = {}) {
  return await db.transaction().execute(async (tx) => {
    const row = await repo.insert(tx, { customerId, createdByAdminId, question });
    const a = baseAudit(ctx);
    await writeAudit(tx, {
      actorType: 'admin',
      actorId: createdByAdminId,
      action: 'question.created',
      targetType: 'customer_question',
      targetId: row.id,
      metadata: { ...a.metadata, customerId },
      visibleToCustomer: true,
      ip: a.ip,
      userAgentHash: a.userAgentHash,
    });

    const recipients = await listActiveCustomerUsers(tx, customerId);
    for (const u of recipients) {
      await recordForDigest(tx, {
        recipientType: 'customer_user',
        recipientId: u.id,
        customerId,
        bucket: 'action_required',
        eventType: 'question.created',
        payload: { questionId: row.id, questionPreview: row.question.slice(0, 120) },
      });
    }
    return row;
  });
}

export async function answerQuestion(db, { id, answeredByCustomerUserId, answerText }, ctx = {}) {
  return await db.transaction().execute(async (tx) => {
    const row = await repo.answer(tx, { id, answeredByCustomerUserId, answerText });
    if (!row) return null;
    const a = baseAudit(ctx);
    await writeAudit(tx, {
      actorType: 'customer',
      actorId: answeredByCustomerUserId,
      action: 'question.answered',
      targetType: 'customer_question',
      targetId: id,
      metadata: { ...a.metadata, customerId: row.customer_id },
      visibleToCustomer: true,
      ip: a.ip,
      userAgentHash: a.userAgentHash,
    });
    const admins = await listAdmins(tx);
    for (const ad of admins) {
      await recordForDigest(tx, {
        recipientType: 'admin',
        recipientId: ad.id,
        customerId: row.customer_id,
        bucket: 'fyi',
        eventType: 'question.answered',
        payload: { questionId: id },
      });
    }
    return row;
  });
}

export async function skipQuestion(db, { id, answeredByCustomerUserId }, ctx = {}) {
  return await db.transaction().execute(async (tx) => {
    const row = await repo.skip(tx, { id, answeredByCustomerUserId });
```

```javascript
import { sql } from 'kysely';

export async function getCleanupBannerForCustomer(db, customerId) {
  const r = await sql`
    SELECT id, ts, metadata
    FROM audit_log
    WHERE action = 'credential.deleted'
      AND actor_type = 'admin'
      AND visible_to_customer = true
      AND metadata->>'customerId' = ${customerId}::text
      AND ts > now() - interval '7 days'
      AND ts > COALESCE(
        (SELECT last_cleanup_banner_dismissed_at FROM customers WHERE id = ${customerId}::uuid),
        '-infinity'::timestamptz
      )
    ORDER BY ts DESC
    LIMIT 1
  `.execute(db);
  if (r.rows.length === 0) return null;
  const row = r.rows[0];
  return {
    auditId: row.id,
    ts: row.ts,
    provider: row.metadata?.provider ?? 'credential',
  };
}

export async function dismissCleanupBanner(db, customerId) {
  await sql`UPDATE customers SET last_cleanup_banner_dismissed_at = now() WHERE id = ${customerId}::uuid`.execute(db);
}
```
